# turbo-memory-mcp 사용 가이드

## Memory MCP 사용 원칙

이 프로젝트에는 `memory` MCP 서버가 등록되어 있다.
중요한 정보, 결정사항, 학습 내용은 반드시 memory에 저장하라.

## Tools

- `remember(texts: [...])` — 배열로 한 번에 저장 (단건보다 배치 우선)
- `recall(query, top_k?)` — 작업 시작 전 관련 기억 먼저 검색
- `forget(id)` — 오래되거나 잘못된 기억 삭제
- `memory_stats()` — 저장 현황 확인

## 언제 저장하는가

- 버그 원인과 해결책
- 설계 결정과 그 이유
- 반복되는 실수 패턴
- 외부 API/라이브러리 사용법 중 비직관적인 것

## 세션 시작 시

새 세션이 시작되면 현재 작업과 관련된 기억을 먼저 recall하라:
```
recall("현재 작업 키워드")
```
